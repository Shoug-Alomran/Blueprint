# Security Policy

Blueprint by ShougTech prioritizes simplicity and secure deployment practices in all projects.

This document outlines how security concerns are handled for Blueprint-managed repositories and client deployments.

---

## Supported Projects

Blueprint focuses exclusively on static website architecture.

Supported project types:
- Digital gardens
- Project documentation systems
- Workshop and course websites

Blueprint does not operate backend systems, databases, authentication systems, or dynamic APIs within this repository.

---

## Security Model

Blueprint projects prioritize:

- Static site architecture (no server-side logic)
- Minimal external dependencies
- HTTPS configuration (via GitHub Pages and/or Cloudflare)
- Clean repository structure
- No embedded secrets or tokens in public repositories

Because Blueprint sites are static by design, the attack surface is intentionally minimized.

---

## Reporting a Vulnerability

If you discover a security vulnerability related to:

- This repository
- A Blueprint-managed deployment
- Domain configuration managed by Blueprint

Please report it privately.

Do NOT open a public issue for security vulnerabilities.

Contact:
Blueprint@Shoug-Tech.com

Reports will be reviewed and addressed as appropriate.

---

## Client Responsibility

If a client chooses to:

- Modify their repository
- Add third-party scripts
- Integrate external services
- Deploy outside Blueprint’s recommended structure

Blueprint cannot guarantee continued security integrity.

Domain ownership and hosting platform policies remain the responsibility of the domain owner.

---

## Disclaimer

Blueprint provides static website deployment services.

Blueprint does not provide cybersecurity audits, penetration testing, or ongoing security monitoring services.

Security best practices are implemented within the scope of static deployment only.
