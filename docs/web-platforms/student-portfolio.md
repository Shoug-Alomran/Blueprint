# Case Study — Student Portfolio Website

## Context

The objective was to design a professional portfolio website that communicates strengths, projects, and outcomes clearly — within two minutes of browsing.

The audience included:

- Instructors  
- Recruiters  
- Technical reviewers  
- Peers  

The site needed to be structured, scannable, and mobile-friendly.

---

## Challenge

<div class="card-grid">

  <article class="info-card">
    <h3>Information Overload</h3>
    <p>Multiple projects with varying depth required clear categorization.</p>
  </article>

  <article class="info-card">
    <h3>Short Attention Span</h3>
    <p>Reviewers typically scan portfolios quickly before deciding to continue.</p>
  </article>

  <article class="info-card">
    <h3>Mobile Readability</h3>
    <p>The portfolio needed to remain clear and structured across devices.</p>
  </article>

</div>

---

## Structural Decisions

<div class="card-grid hero-cards">

  <article class="hero-card">
    <h3>Summary-First Homepage</h3>
    <p>A concise overview highlighting strengths and featured projects immediately.</p>
  </article>

  <article class="hero-card">
    <h3>Categorized Project Pages</h3>
    <p>Projects grouped by theme or complexity for easier evaluation.</p>
  </article>

  <article class="hero-card">
    <h3>Minimal About & Contact</h3>
    <p>Clear biography and contact path without unnecessary clutter.</p>
  </article>

</div>

---

## Build Scope

- Static responsive layout  
- Lightweight navigation system  
- Clear typography and spacing hierarchy  
- GitHub Pages deployment  
- Long-term maintainable structure  

Built using static architecture to ensure speed, reliability, and minimal upkeep.

## Why This Structure Worked

A predictable page hierarchy reduced cognitive load.

By prioritizing summary, categorization, and clarity over visual complexity, the site supported fast evaluation — which is critical for academic and early-career portfolios.

Structure created credibility.
