# Case Study — Workshop Course Page

## Context

The project required a participant-facing workshop website that clearly communicated:

- Session schedule  
- Required materials  
- Participation guidelines  
- Learning outcomes  

The goal was to reduce confusion and eliminate repetitive logistical questions before session day.

---

## Challenge

<div class="card-grid">

  <article class="info-card">
    <h3>Information Overload</h3>
    <p>Workshop details were scattered across messages and documents.</p>
  </article>

  <article class="info-card">
    <h3>Participant Confusion</h3>
    <p>Attendees repeatedly asked for schedule and material clarification.</p>
  </article>

  <article class="info-card">
    <h3>Time Sensitivity</h3>
    <p>Content needed to be accessible quickly across devices.</p>
  </article>

</div>

---

## Structural Decisions

<div class="card-grid hero-cards">

  <article class="hero-card">
    <h3>Agenda-First Homepage</h3>
    <p>The schedule was placed at the top to answer the most common question immediately.</p>
  </article>

  <article class="hero-card">
    <h3>Dedicated Resources Section</h3>
    <p>Materials were centralized into one organized section for easy access.</p>
  </article>

  <article class="hero-card">
    <h3>Single-Flow FAQ</h3>
    <p>Participation details were structured in one clear, scroll-based flow.</p>
  </article>

</div>

---

## Build Scope

- Static workshop website  
- Responsive schedule layout  
- Structured section cards  
- Clear navigation hierarchy  
- Launch-ready deployment setup  

Built using static architecture for speed and reliability.

## Why This Structure Worked

Organizing information around the participant journey — from registration to attendance — created clarity.

The agenda-first layout answered primary concerns immediately, while centralized resources eliminated scattered communication.

Structured hierarchy reduced friction and improved usability without adding technical complexity.
