## كيف يعرف برينت الهيكلة

**القسم**
عنصر تنقل رئيسي داخل الموقع مثل نظرة عامة أو منهجية أو نتائج.

**الصفحة**
صفحة محتوى مستقلة داخل قسم معين.

مثال:
إذا لديك قسم منهجية وداخله صفحتان تفصيليتان، فهذا يعني:

- قسم واحد
- صفحتان

الأقسام تحدد التنقل. الصفحات تحدد عمق المحتوى.

---

## مرونة التحويل من قسم إلى صفحة

كل باقة تشمل عددا محددا من الأقسام.

إذا كان مشروعك يحتاج أقساما أقل، يمكن تحويل الأقسام غير المستخدمة إلى صفحات إضافية بدون تغيير سعر الباقة.

يتم تأكيد نسبة التحويل كتابيا أثناء مراجعة النطاق قبل بدء التنفيذ.

### سياسة الصيانة

الصيانة الشهرية تشمل تحديثات محتوى مستمرة وتعديلات بسيطة ضمن الهيكل الحالي.

إذا تطلب المشروع قسما جديدا، يتم احتسابه بسعر القسم الإضافي.

إعادة التصميم الكامل أو تغيير هيكل الباقة يتم تسعيره بشكل مستقل.

## ما لا يشمله النطاق {#what-is-not-included}

خدمات برينت لا تشمل:

- تطبيقات Full-stack
- قواعد البيانات
- أنظمة تسجيل الدخول والمصادقة
- تطوير منصات SaaS
- تكاملات Backend معقدة
- تطبيقات ويب مخصصة

أي مشروع يحتاج أنظمة ديناميكية يكون خارج نطاق الخدمة.

## ملاحظات

- مدة التسليم تعتمد على جاهزية المحتوى
- تأخر الملاحظات قد يمدد الوقت
- الطلبات خارج النطاق تسعر بشكل مستقل
- رسوم الاستضافة والدومين منفصلة ما لم يذكر خلاف ذلك
- تحويل التقرير إلى HTML يتطلب استلام الملف كاملا قبل البدء
- تحويل الأقسام إلى صفحات يتم تأكيده أثناء مراجعة النطاق
