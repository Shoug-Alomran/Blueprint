## ليه تشتغل مع برينت؟
 **تنقل منظم** عشان الزوار يلقون اللي يبونه بسرعة، **هندسة ثابتة بالأساس** عشان السرعة والاستقرار وصيانة أقل، **عرض نظيف** يعطي انطباع احترافي (مو "قالب طلابي" معروف)، **تسليم سهل** عشان تقدر تحدث المحتوى بنفسك بدون لا تكسر الموقع  

## حدود الشغل

| يدخل في الشغل | ما يدخل في الشغل |
| --- | --- |
| مواقع ويب ثابتة (MkDocs / GitHub Pages / Cloudflare Pages) | تطبيقات متكاملة (باك إند + فرونت إند) |
| تنظيم المحتوى وهيكلة المعلومات | قواعد بيانات |
| واجهة مستخدم متجاوبة + أساسيات سهولة الوصول | أنظمة تسجيل دخول |
| أساسيات تحسين محركات البحث (عناوين، وصف، فهرسة) | بناء منتجات برمجية معقدة (SaaS) |
| تجربة النشر + اختبار + قائمة تسليم نهائية | رسوم متحركة معقدة حسب الطلب أو تطبيقات ويب ثقيلة |
