# 03 · الهيكلة وخريطة الموقع

هذه المرحلة تحدد كيف يتنقل الزائر داخل الموقع. يتم اعتماد تسلسل التنقل وتجميع المحتوى بشكل نهائي قبل أي تصميم أو تنفيذ واجهة.

---

## ما الذي يتم تصميمه

<div class="card-grid hero-cards">

  <article class="hero-card">
    <h3>تسلسل الأقسام</h3>
    <p>تحديد كل الأقسام الرئيسية والصفحات التابعة لكل قسم.</p>
  </article>

  <article class="hero-card">
    <h3>نظام التنقل</h3>
    <p>تحديد مسارات التنقل الأساسية وأي تنقل ثانوي أو جانبي.</p>
  </article>

  <article class="hero-card">
    <h3>تجميع المحتوى</h3>
    <p>تجميع المحتوى المتشابه لتقليل الحمل المعرفي وتسريع الفهم.</p>
  </article>

  <article class="hero-card">
    <h3>تدفق المستخدم</h3>
    <p>رسم مسار انتقال الجمهور المستهدف من نقطة الدخول إلى المحتوى الأهم.</p>
  </article>

</div>

## مخرجات هذه المرحلة

- خريطة موقع معتمدة  
- تسلسل تنقل نهائي  
- خريطة محتوى جاهزة للبناء  

يتم اعتماد الهيكلة من العميل قبل بدء البناء. أي تغيير هيكلي بعد الاعتماد يُراجع ويُسعّر بشكل مستقل.
